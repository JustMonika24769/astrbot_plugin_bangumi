# Changelog

## v1.5.0

### 变更
- **默认卡片风格**: `episode_card_template` 默认值调整为 `pastel_lightbox`,搜索结果卡片、单集更新卡片和长文本响应卡片默认使用 Pastel 版。

### 修复
- **搜索结果简介完整渲染**: Pillow 条目详情卡不再按旧的三行简介高度截断,中日文或英文长简介会按实际文本高度参与整卡排版。
- **多集数进度渲染**: 条目详情卡的放送进度网格改为 7 个一行,28 集可完整显示为 4 行,避免超过 15/18 集后缺失。

### 优化
- **条目卡动态高度**: 搜索结果卡会在左侧封面、集数和评分分布高度与右侧简介高度之间取更合适的布局高度;短简介时 footer 跟随左侧评分分布底部附近,长简介时随正文向下扩展。

## v1.4.2

### 新增
- **内置指令帮助**: `/bgm` 不带关键词或使用 `/bgm help` 时会直接返回常用指令清单,新用户不用再翻 README 才能知道怎么搜索、追番、取消订阅或切换卡片风格。

### 修复
- **分类搜索别名**: `/bgm动漫`、`/bgm动画`、`/bgm番`、`/bgm动画片` 和 `/bgm电影` 改为 AstrBot 4.26.2 原生别名注册方式,避免分类搜索被当成普通 `/bgm` 查询或无法被正确捕获。

### 优化
- **卡片排版可读性**: 搜索结果卡片和单集卡片优化了标签、徽章、集数和长标题的居中、换行与截断表现,减少文字贴边、偏移或挤在一起的情况。
- **指令文档提示**: README 补充 `/bgm help` 用法,并明确分类搜索命令需要使用 `/bgm番剧` 这类连续形式。

## v1.4.1

### 新增
- **搜索结果三风格**: `/bgm`、`/bgm番剧`、`/bgm剧场版`、`/bgm漫画` 的条目搜索结果卡片同步支持 `pastel_lightbox`、`editorial_digest`、`cinematic_poster` 三种风格,并复用 `/bgm模板` 和 `episode_card_template` 配置。
- **搜索卡预览**: 新增真实数据搜索结果卡片预览脚本,用于审核三种风格在真实 Bangumi 条目上的可读性和差异度。
- **单集简介自动翻译**: 新增 `auto_translate_episode_summary` 配置,订阅更新渲染单集卡片前可调用 AstrBot 默认聊天模型将非空单集简介翻译为中文;无默认模型、返回空文本或翻译失败时保留 Bangumi 原简介。
- **AstrBot 版本要求**: 插件元数据最低 AstrBot 版本调整为 `>=4.26.2`,与订阅更新主动发送消息能力保持一致。

## v1.4.0

### 新增
- **长文本响应图片化**: 30 字以内且无换行的响应继续使用纯文字;更长或包含换行的响应会按当前卡片风格渲染为图片,渲染失败时退回原纯文字。
- **三渲染模式**: `render_mode` 支持 `pillow`、`playwright`、`rpc`,默认使用 `pillow`;旧配置值 `html` 会兼容映射为本地 Playwright。
- **得意黑字体预热**: 插件启动时优先检测本地得意黑/Smiley Sans 字体缓存,不存在则后台拉取官方发布包,失败时退化到现有默认字体链路。
- **响应卡预览**: 新增真实数据响应卡预览脚本,用于在用户审核前生成三种风格的可读性检查图。

## v1.3.2

### 新增
- **搜索指令别名**: `/bgm番剧` 现在支持 `/bgm动漫`、`/bgm动画`、`/bgm番`、`/bgm动画片` 同义触发;`/bgm剧场版` 现在支持 `/bgm电影` 同义触发。

## v1.3.1

### 修复
- **追番候选确认**: `/追番` 多候选确认阶段支持直接回复裸数字序号,例如 `1`;非法输入时会明确提示可用格式,避免静默等待导致空回复。

## v1.3.0

### 架构与渲染
- **结构分层**: 新增 `src/domain`、`src/api`、`src/app` 分层,保留 `src.services` 旧路径兼容导出
- **本地策略分析**: 新增只读脚本 `python scripts/analyze_bangumi_strategy.py`,用于本地采集 Bangumi 收藏与可选 AstrBot SQLite 订阅数据并生成策略分析报告,不影响插件运行时行为
- **单集卡片变体**: 单集更新卡片保留 `pastel_lightbox`、`editorial_digest`、`cinematic_poster` 三种模板,默认使用第三个 `cinematic_poster`,并支持通过 `episode_card_template` 或 `/bgm模板` 切换
- **单集卡片预览**: 新增 `python scripts/render_episode_variants.py`,默认使用 Bangumi API 真实条目与剧集数据生成三种卡片预览图,并支持 HTML/Pillow 像素对齐报告
- **Pillow 渲染退避**: 条目、单集、放送表在 `render_mode=pillow` 下使用纯 Pillow 出图,避免回落到 Playwright
- **默认渲染链路**: 默认按 RPC -> Playwright -> Pillow 顺序自动退避,并在初始化时后台预热 Pillow 字体,避免阻塞插件启动
- **Pillow/Playwright 对齐**: 条目、单集、放送表最终尺寸、宽高比与 alpha 均通过对齐校验,显著变化像素分别为 6.115%、3.038%、4.203%;最终证据目录为 `rendered_images/render-mode-comparison-2026-05-16-group-1-final`,精确字形/浏览器栅格一致性仍作为已知实践限制
- **渲染健壮性**: 补充 CJK 字体候选、空白封面占位图检测和图片读取大小限制,提升卡片可读性与安全性
- **渲染测试补强**: 补齐本地 Playwright 截图冒烟测试,并为 Pillow 输出补充 PNG 格式、尺寸与非空白断言
- **配置辅助**: 支持从 `_conf_schema.json` 生成本地测试 `.env`,并默认保留已有填写值
- **代理配置修复**: 未配置代理时不再隐式启用 `127.0.0.1:7890`;已配置代理会自动去除首尾空白,省略 scheme 时补全 `http://`,并保留已有 scheme/端口
- **访问令牌清理**: `access_token` 会先去除首尾空白;空白令牌不再生成 `Authorization` 请求头
- **取消订阅清理**: `/弃坑` 删除最后一个订阅时同步清理孤立的 `BangumiSubject`,多个群仍订阅同一条目时保留条目记录
- **运行时回归覆盖**: 补充代理、令牌与订阅清理的聚焦回归测试,确认旧 refactor 分支仅移植必要硬化行为
- **质量门禁**: 补齐 API、app、db、render、config、utils 与主插件入口测试,并修复严格 mypy 检查问题

- **取消订阅匹配**: 修复本群订阅候选的模糊匹配回退逻辑,支持命中更接近的条目名称

## v1.1.1

### 功能拓展
- **搜索优化**: `/追番` 现在支持候选确认:当搜索到多个结果时,会先返回列表,可通过 `/追番 序号` 选择目标,减少订错番剧的情况
- **取消订阅优化**: `/弃坑` 的匹配更贴近群聊使用场景:优先在本群已订阅列表中匹配,取消订阅更准确

## v1.1.0

### 新增功能
- **取消订阅**: 新增 `/弃坑` 命令,支持群组移除已订阅的番剧更新提醒
- **更新卡片渲染**: 引入 `EpisodeRenderer`,在番剧更新时自动推送精美的单集图文通知卡片
- **命令别名**: 简化常用命令,支持使用 `/bgm` 代替 `/bgm搜索`

### 核心优化
- **更新检测逻辑**: 重构剧集更新判定算法,结合播出日期与评论互动数据(`comment > 0`),显著降低更新误报率
- **全链路 Base64 渲染**: 渲染引擎(放送表、剧集卡片)全面转向 Base64 内存流,移除临时文件 IO,提升并发性能
- **Playwright 鲁棒性**: 优化浏览器安装与初始化逻辑,支持非交互式环境安装,并提供实时状态日志
- **类型系统增强**: 引入完整的 `SubjectType`、`ImageSize` 等枚举类型,提升代码可维护性
- **代码重构**: 优化 `SubjectsService` 的数据解析流,通过 Pydantic 严格过滤异常 API 返回

## v1.0.0

### 新增功能
- **分类搜索**: 新增 `/bgm番剧`、`/bgm剧场版`、`/bgm漫画` 命令,支持更精准的类型过滤
- **每日放送**: 新增 `/today` 命令,渲染精美的每日番剧放送表图片
- **追番系统**: 新增 `/追番` 功能,支持订阅番剧并在有新集数更新时自动向群组推送通知
- **通用搜索优化**: `/bgm搜索` 命令现在支持更完善的参数处理和 top_k 结果返回

### 代码优化
- **渲染引擎重构**: 引入 `SubjectRenderer` 和 `CalendarRenderer`,基于 Playwright 实现更美观的图文卡片
- **数据库集成**: 引入 SQLAlchemy 驱动的 SQLite 存储,用于管理番剧信息和订阅关系
- **自动更新逻辑**: 新增定时任务,每小时自动检查订阅番剧的更新状态
- **重构逻辑**: 将搜索与渲染逻辑分离,提取出 `_render_subjects` 和 `_handle_subject` 核心方法,提高代码复用性
- **修复 Bug**: 修复了搜索命令中生成器未正确迭代导致无响应的问题
- **类型提示**: 为核心方法添加了完善的类型注解和文档说明
